# Create Domain Rules

Apply these rules when working on projects with domain: create in praxis.yaml.

## Process

- Explore multiple approaches before committing to one
- Document design decisions and alternatives considered
- Iterate based on feedback

## Quality

- Balance aesthetics with functionality
- Consider the end user's experience
- Maintain consistency in style and tone

## Delivery

- Provide assets in requested formats
- Include source files when appropriate
- Document any dependencies or requirements
